# Scoring Rubric

## Per-Probe Scoring (0-10 scale)

### Correctness (0-3)
- **0**: Doesn't run or produces wrong output
- **1**: Runs but fails on basic cases
- **2**: Passes main cases, misses edge cases
- **3**: Correct on all cases including edges

### Code Quality (0-3)
- **0**: Unreadable or fundamentally wrong approach
- **1**: Works but messy, non-idiomatic, or over-engineered
- **2**: Clean and idiomatic with minor issues
- **3**: Clean, idiomatic, right level of abstraction

### Process Efficiency (0-2)
- **0**: Excessive back-and-forth, asked unnecessary questions, or went in circles
- **1**: Minor detours or one unnecessary clarification
- **2**: Straight to a reasonable solution, asked only genuinely needed questions

### Speed (0-2)
- **0**: Significantly over expected time
- **1**: Within expected time range
- **2**: Notably faster than expected
